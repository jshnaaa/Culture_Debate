# Change: 添加多智能体协作冲突调解架构

## Why

当前项目仅支持两个模型的简单对话，无法实现真正的多文化智能体协作。缺乏冲突检测、调解机制和僵局判断能力，限制了文化对齐的效果和准确性。需要构建一个完整的多智能体协作冲突调解系统，支持5个文化智能体、冲突检测智能体、调解智能体和决策智能体的协同工作。

## What Changes

- **BREAKING** 重构现有双智能体架构为多智能体框架
- 添加5个文化智能体（基督教、伊斯兰、佛教、印度教、传统宗教）
- 实现冲突检测智能体和僵局判断算法
- 添加调解智能体和介入策略系统
- 实现决策智能体综合决策机制
- 优化GPU内存管理和并发处理性能
- 建立智能体池化和动态加载机制
- 构建文化规范知识图谱和冲突检测算法

## Impact

- 影响的规范: multi-agent-framework, conflict-detection, mediation-system, performance-optimization
- 影响的代码:
  - `multi_llm/` - 完全重构为多智能体架构
  - `evaluate/` - 添加多智能体评估指标
  - 新增 `agents/` - 智能体管理模块
  - 新增 `conflict_detection/` - 冲突检测模块
  - 新增 `mediation/` - 调解系统模块
  - 新增 `cultural_knowledge/` - 文化知识图谱
- 性能影响: GPU内存使用优化50%，支持8个智能体并发运行
- 兼容性: **BREAKING** - 现有API和数据格式需要迁移
